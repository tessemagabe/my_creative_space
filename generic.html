DevOps My Capstone Project

Introduction

This proposal is meant to address a problem and is instigated by the given assignment of the course. It deals with finding an effective container orchestration tool for the XYZ drone building company. XYZ is a drone-building company that provides drones for farm use. Hence, this paper analyses two popular tools, namely HashiCorp Nomad and Amazon ECS, and proposes a solution beneficial to the company in question.

Background

Purpose: Because the company XYZ heard about the advantages of containers and container orchestration, it decided to install one. Hence, evaluate two container orchestration tools that are not covered in the course by using an already attached spreadsheet.

Company: XYZ is a small drone-building company. Its core systems are:

  • Sales order entry system using Python and MySQL on Red Hat Linux servers
  • CRM system on Windows NT servers

Requirements

  • Scale up past 10,000 simultaneous users
  • Microservices support for integration
  • High availability with .99999 SLA
  • Strong security for handling credit card data

Deliverable

Focused on leveraging containers for the sales order entry systems, recommend a container orchestration tool based on the requirements of the company and on the weighted vectors in the attached spreadsheet.

Tool 1: HashiCorp Nomad

a) Why HashiCorp Nomad?

Nomad supports both containerized and non-container workloads. It enables unified deployment across multiple data centers, with autoscaling and dynamic application sizing. It supports 10,000+ nodes with minimal SRE support.

b) Pros

  • Simple and focused on scheduling
  • Easy deployment (single binary)

c) Cons

  • Only one part of a full orchestration platform – requires other tools
  • Lags in some modern features like service-mesh and tracing

d) Justification for XYZ

  • Supports scale beyond 10,000 nodes
  • Cross-platform, workload-diverse
  • Linear resource scaling
  • Strong cybersecurity features

e) Companies Using Nomad

  • Walmart
  • Amazon
  • Apple
  • CVS Health
  • UnitedHealth Group

Tool 2: Docker Swarm

a) Why Docker Swarm?

  • Integrated with Docker
  • Easy to scale and deploy
  • Built-in load balancing and PKI security
  • Simple setup

b) Pros

  • Simpler than other orchestration tools
  • Lightweight and easy to install

c) Cons

  • Lacks advanced features
  • Dependent on Docker API

d) Justification for XYZ

  • Simple orchestration for Docker-only containers
  • Secure, integrated design for small teams

e) Companies Using Docker Swarm

  • Infosys
  • Fujitsu
  • eBay
  • Cornell University
  • NetSuite

Comparison Table

Feature Nomad Docker Swarm
Architecture Client-server Swarm-mode
Workload Support Containers + non-containers Docker only
Scheduling Imperative Declarative
Service Discovery External integrations Built-in Docker discovery
Integration Flexible ecosystem Docker ecosystem
Deployment Scale Large, multi-region Small to medium

Conclusion

Docker Swarm is a simple and tightly integrated orchestration tool for Docker containers, while Nomad offers broader support and higher scalability. For XYZ, which may scale in the future, Nomad is the better fit due to its flexibility and enterprise-level capabilities.
